This task demonstrates basic data exploration and data cleaning using Python and Pandas. The dataset used is an ecommerce shopping dataset containing information about products, prices, ratings, reviews, and categories.

The goal of this task is to understand how raw data can be cleaned, transformed, and prepared for analysis before being used for reporting, visualization, or machine learning.

---


## Objective

The objectives of this task are:

1. Load a CSV dataset into a Pandas DataFrame.
2. Explore the dataset structure and contents.
3. Handle missing values (identify, fill, and drop).
4. Perform basic operations such as filtering rows and selecting columns.
5. Remove duplicate records.
6. Create a derived column (`total_amount = final_price × quantity`).
7. Save the cleaned dataset as a new CSV file.

# Project Workflow

## 1. Load the Dataset

The dataset was loaded into a Pandas DataFrame using:

pd.read_csv()

The dataset was successfully imported and the number of rows and columns was verified.

---

## 2. Data Exploration

The following functions were used to understand the dataset:

- head()
- tail()
- shape
- columns
- info()
- describe()

### Purpose

- Understand dataset structure
- Check column names
- Analyze data types
- Generate statistical summaries
- Identify missing values

---

## 3. Handle Missing Values

Missing values were identified and handled using appropriate techniques.

### Actions Performed

- Missing ratings were filled using the median value.
- Missing ratings count values were filled using the median value.
- Rows with missing product titles were removed.

### Methods Used
fillna()
dropna()
---

## 4. Perform Basic Operations

Relevant columns were selected for analysis.

### Selected Columns

- product_id
- title
- initial_price
- final_price
- rating
- ratings_count

### Filtering

Products with ratings greater than or equal to 4 were retained:
df[df['rating'] >= 4]

This allowed the analysis to focus on highly rated products.

---

## 5. Remove Duplicate Records

Duplicate rows were removed to improve data quality and avoid inaccurate results.

### Method Used

drop_duplicates()

Benefits:

- Improved data consistency
- Better statistical accuracy
- Reduced redundancy

---

## 6. Create Derived Columns

Two new columns were created for analysis.

### Quantity

Random values between 1 and 5 were generated to simulate product purchases.

### Total Amount

Calculated using:
total_amount = final_price * quantity

### Purpose

This helps estimate revenue generated by each product.

## 7. Save the Cleaned Dataset

The cleaned dataset was exported as a new CSV file.

### Method Used

to_csv(index=False)
### Output File
cleaned_dataset.csv

## Key Insights

Some observations from the dataset:

- Products with ratings greater than or equal to 4 were selected for analysis.
- Price columns required cleaning before numerical operations could be performed.
- Missing values were present in a few columns and were handled appropriately.
- Derived columns such as `quantity` and `total_amount` helped simulate basic revenue analysis.
- Filtering reduced the dataset size and allowed focus on highly rated products.

- ## Challenges Faced

During the project, the following challenges were encountered:

- Price values were stored as text and required cleaning before analysis.
- Missing values needed different handling strategies depending on the column type.
- Some columns contained complex text and nested information that were not relevant to the assignment.
- Selecting the most useful columns required understanding the dataset structure.
- Managing data types correctly was important for calculations and filtering.

- ## Limitations

- The dataset does not contain actual purchase quantity information.
- Quantity values were simulated for learning purposes.
- Revenue calculations are estimates and not real business revenue.
- Only basic data cleaning and exploration techniques were applied.
- Advanced analytics and visualizations were outside the scope of this assignment.
